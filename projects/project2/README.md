# 프로젝트 2 — 공정 데이터 분석기

> **렛유인 AI + 바이브코딩 강의** | 10차시 이후 주말 프로젝트 (2일)

---

## 개요

반도체 · 디스플레이 · 2차전지 등 실제 제조 공정에서 수집되는 센서/품질 데이터를 분석하고,
이상 징후를 시각화하는 **대화형 웹 앱**을 만드는 프로젝트입니다.

Gemini AI에게 요청하면서 코드를 만들어 가는 **바이브코딩** 방식으로 진행합니다.
완성된 앱은 Streamlit Cloud 또는 로컬에서 바로 실행 가능합니다.

---

## 사용 기술

| 도구 | 역할 |
|------|------|
| **Streamlit** | 웹 앱 UI 프레임워크 |
| **Plotly** | 인터랙티브 차트 |
| **Pandas / NumPy** | 데이터 처리 |
| **SciPy** | 통계 분석 |
| **Gemini API** | AI 인사이트 생성 |

---

## 난이도 선택

```
project2/
├── examples/
│   ├── 하/   <- 단계별 가이드 + 힌트 제공
│   ├── 중/   <- 일부 힌트, 프롬프트 직접 작성
│   └── 상/   <- 힌트 없음, 전체 설계 직접 수행
```

### 하 — 기초 대시보드

- SECOM 스타일 합성 데이터 (온도, 압력, 두께, 수율)
- Plotly 라인 차트 · 히스토그램 · 기초 통계 테이블
- 사이드바 필터 (날짜 범위, 공정 라인 선택)
- Gemini 프롬프트 예시 제공

**목표:** Streamlit 앱 구조 이해 + 기본 시각화

---

### 중 — 파라미터 조절 + AI 인사이트

- 50개 센서 x 1000 샘플 합성 데이터
- 상관관계 히트맵, 산점도 행렬
- 사이드바에서 센서 선택 · 임계값 조절
- SPC 관리도 (X-bar, UCL/LCL)
- "AI 분석" 버튼 -> Gemini API 연동 인사이트

**목표:** API 연동 + 프롬프트 직접 설계

---

### 상 — 자동 이상 감지 + 최적 조건 추천

- IQR / Z-score 기반 자동 이상 감지
- Pareto 차트 (불량 원인 분석)
- 최적 파라미터 조건 추천
- 멀티 탭 Streamlit (Overview / 이상 감지 / 최적화 / 리포트)
- 히트맵 시각화 (웨이퍼 맵 스타일)
- Markdown 리포트 자동 생성 + 다운로드
- Streamlit Cloud 배포

**목표:** 엔드-투-엔드 분석 시스템 설계

## 시작하기

### 1. 환경 설정

```bash
cd examples/하   # 또는 중, 상
pip install -r requirements.txt
```

### 2. API 키 설정 (중·상 난이도)

```bash
cp .env.example .env
# .env 파일에 Gemini API 키 입력
```

### 3. 앱 실행

```bash
streamlit run app.py
```

## Gemini 바이브코딩 프롬프트 예시

### 하 난이도 시작 프롬프트

```
나는 Streamlit으로 공정 데이터 대시보드를 만들려고 해.
합성 데이터를 생성해서 온도, 압력, 두께, 수율을 시각화하고 싶어.
Plotly로 라인 차트와 히스토그램을 그려줘.
사이드바에 날짜 필터도 추가해줘.
```

### 중 난이도 시작 프롬프트

```
50개 센서에서 1000개 샘플을 수집한 공정 데이터를 분석하는 앱을 만들어줘.
상관관계 히트맵과 SPC 관리도가 필요해.
Gemini API로 이상 패턴을 설명하는 인사이트를 생성하는 버튼도 추가해줘.
```

### 상 난이도 시작 프롬프트 (스스로 설계)

```
제조 공정 이상 감지 시스템을 처음부터 설계해.
IQR과 Z-score로 이상치를 탐지하고, Pareto 차트로 불량 원인을 분석하고,
Gemini API로 최적 파라미터를 추천하는 멀티탭 앱을 만들어.
```
