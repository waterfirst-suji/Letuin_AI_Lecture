Git이 뭔데 이렇게 중요해?

“파일 이름에 _최종, _최종2, _진짜최종 붙여본 적 있나요?”

그게 바로 Git이 해결하는 문제다. Git은 파일의 모든 변경 이력을 추적한다.

Git 핵심 개념 3가지만 알면 된다

commit

변경사항을 저장하는 스냅샷.

“이 시점의 코드를 기록해둬”

git add .
git commit -m "수율 분석 추가"

push / pull

로컬 → GitHub 업로드 (push)

GitHub → 로컬 다운로드 (pull)

git push origin main
git pull origin main

branch

실험적 기능을 안전하게 개발.

메인 코드에 영향 없이 테스트.

git checkout -b new-feature
git merge new-feature

실무에서 70%는 이 세 가지만 쓴다. 나머지는 필요할 때 AI한테 물어보면 된다.

Claude Code 설치 — AI 코딩 어시스턴트를 터미널에

Claude Code는 터미널에서 동작하는 AI 코딩 어시스턴트다. 프로젝트 전체를 이해하고, 파일을 직접 수정한다.

설치 순서

# Step 1: Node.js LTS 설치 확인
node --version
# v18 이상이어야 함

# Step 2: Claude Code 전역 설치
npm install -g @anthropic-ai/claude-code

# Step 3: 설치 확인
claude --version

# Step 4: 프로젝트 폴더에서 첫 실행
cd my-display-project
claude

Node.js 없으면?

nodejs.org → LTS 버전 다운로드 → 설치

설치 후 터미널(PowerShell) 재시작 필수!

Windows: winget install OpenJS.NodeJS

Mac: brew install node

Claude Code = 전 세계 개발자들이 가장 많이 쓰는 AI 코딩 도구 중 하나. GitHub Stars 수만 개.

CLAUDE.md — AI의 두뇌를 프로젝트에 세팅하라

CLAUDE.md란?

Claude Code가 프로젝트를 시작할 때 가장 먼저 읽는 파일.

여기에 “이 프로젝트는 디스플레이 수율 분석 앱이야”라고 쓰면 AI가 맥락을 이해한다.

매번 설명하지 않아도 된다. 한 번 써두면 영원히 작동한다.

디스플레이 엔지니어 프로젝트용 CLAUDE.md 예시

# 프로젝트: 삼성 디스플레이 수율 분석 포트폴리오

## 배경
이 프로젝트는 삼성 디스플레이 취업을 위한 포트폴리오입니다.
OLED/LCD 패널 수율 데이터를 분석하고 시각화합니다.

## 기술 스택
- Python 3.11 + pandas + matplotlib
- R + ggplot2 (데이터 시각화)
- GitHub Pages (배포)

## 코딩 규칙
- 한국어 주석 사용
- 함수마다 docstring 작성
- 수율(Yield Rate) = 양품 수 / 전체 생산 수 * 100

## 금지 사항
- 실제 삼성 내부 데이터 사용 금지 (가상 데이터만)

GitHub Pages로 배포 — 무료 호스팅

왜 GitHub Pages인가?

  • 완전 무료 — 영구 무료 호스팅
  • 자동 배포 — push하면 바로 반영
  • 취업에 유리 — 포트폴리오 URL이 곧 GitHub URL
  • 커스텀 도메인 — 나중에 연결 가능

배포 URL 형식

https://[사용자명].github.io/[저장소명]/

예시:

https://park-display.github.io/yield-analyzer/

배포 설정 (단 3단계)

① Settings → Pages

GitHub 저장소 → Settings → Pages

② Source 선택

Deploy from a branch → main → /(root)

③ Save 클릭

2분 후 URL 자동 생성!

GitHub 프로필 — 취업에 직결되는 포트폴리오

삼성/LG 채용 담당자가 보는 것

  1. Contribution 잔디밭 — 꾸준함을 보여줌
  2. README.md 품질 — 프로젝트 설명 능력
  3. 코드 품질 — 주석, 구조, 가독성
  4. 스타 수 — 다른 개발자들의 인정
  5. 배포된 URL — 실제로 동작하는가
