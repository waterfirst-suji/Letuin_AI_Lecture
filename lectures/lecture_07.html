Quarto란?

RMarkdown의 진화형

RMarkdown을 알고 있다면? Quarto는 그 다음 세대입니다. 모르더라도 괜찮습니다 — 오늘부터 시작하면 됩니다.

하나의 .qmd 파일 → 여러 출력 형식:

  • HTML 보고서
  • PDF 문서
  • RevealJS 슬라이드 (지금 이것!)
  • 웹사이트 / 블로그
  • Word 문서
  • 학술 논문 (LaTeX)

작동 방식

.qmd 파일
    │
    ▼
knitr / jupyter
(코드 실행)
    │
    ▼
Pandoc
(문서 변환)
    │
    ▼
HTML / PDF / RevealJS

quarto render report.qmd 명령 하나로 전체 파이프라인 실행.

.qmd 파일 구조 해부

---                          ← YAML 헤더 (출력 형식, 제목, 저자 설정)
title: "월간 수율 분석 보고서"
author: "공정팀 김엔지니어"
date: today
format: html
---

핵심 차이점

기존 방식에서 “데이터가 바뀌었습니다”는 반나절~하루의 재작업을 의미합니다. Quarto에서는 터미널에서 quarto render 입력 후 커피 한 잔 마시면 됩니다.

실습 — 디스플레이 공정 단계별 불량률 분포

공정 분석 보고서에 바로 넣을 수 있는 차트:

library(ggplot2)

set.seed(99)
process_data <- data.frame(
  Step = rep(c("포토", "식각", "증착", "세정", "검사"), each = 30),
  DefectRate = c(
    rnorm(30, 2.1, 0.3), rnorm(30, 1.8, 0.4),
    rnorm(30, 3.2, 0.6), rnorm(30, 0.8, 0.2),
    rnorm(30, 0.5, 0.1)
  )
)

ggplot(process_data, aes(x = Step, y = DefectRate, fill = Step)) +
  geom_boxplot(alpha = 0.7, outlier.color = "#E74C3C") +
  theme_minimal(base_size = 13) +
  scale_fill_brewer(palette = "Set2") +
  labs(
    title = "공정 단계별 불량률 분포",
    subtitle = "증착 공정에서 불량률 가장 높고 변동성도 큼 — 집중 개선 필요",
    y = "불량률 (%)", x = "공정 단계"
  ) +
  theme(legend.position = "none")

디스플레이 불량 원인 파레토 차트

어느 원인부터 잡아야 할까? 파레토 법칙: 상위 20% 원인이 80% 문제를 만든다.

library(ggplot2)
library(dplyr)

defects <- data.frame(
  Cause = c("이물질", "스크래치", "얼라인불량", "색얼룩", "단선", "기타"),
  Count = c(45, 32, 28, 19, 12, 8)
) %>%
  arrange(desc(Count)) %>%
  mutate(
    Cause = factor(Cause, levels = Cause),
    CumPct = cumsum(Count) / sum(Count) * 100
  )

ggplot(defects, aes(x = Cause)) +
  geom_bar(aes(y = Count), stat = "identity", fill = "#2E75B6", width = 0.6) +
  geom_line(aes(y = CumPct * max(Count) / 100, group = 1),
            color = "#E74C3C", linewidth = 1.2) +
  geom_point(aes(y = CumPct * max(Count) / 100),
             color = "#E74C3C", size = 3) +
  geom_hline(yintercept = 0.8 * max(defects$Count),
             linetype = "dashed", color = "gray50") +
  scale_y_continuous(
    name = "불량 건수",
    sec.axis = sec_axis(~ . / max(defects$Count) * 100, name = "누적 비율 (%)")
  ) +
  theme_minimal(base_size = 13) +
  labs(
    title = "디스플레이 불량 원인 파레토 차트",
    subtitle = "이물질 + 스크래치 + 얼라인불량 = 전체 불량의 73% — 이 3가지가 핵심",
    x = "불량 원인"
  )

코드 청크 옵션 마스터하기

```{r}
#| label: yield-trend-chart      # 차트 ID (교차 참조에 사용)
#| echo: false                   # 코드 숨김, 차트만 출력
#| fig-width: 10                 # 너비 설정
#| fig-height: 5                 # 높이 설정
#| fig-cap: "라인별 월간 수율 트렌드"  # 캡션
#| cache: true                   # 결과 캐싱 (느린 계산에 유용)

library(ggplot2)
# 차트 코드...
```


자주 쓰는 옵션:

옵션 값 의미
echo true/false 코드 표시 여부
eval true/false 코드 실행 여부
warning true/false 경고 표시
cache true/false 결과 캐싱

보고서용 권장 전역 설정:

execute:
  echo: false
  warning: false
  message: false
  cache: true

모든 청크에 개별로 쓰는 대신 YAML 헤더에 한 번에 설정.

인라인 코드 — 숫자 오타 방지

보고서에서 숫자가 틀리는 가장 큰 이유: 수동 입력.

기존 방식 (위험):

이번 달 평균 수율은 96.2%이며
전월 대비 0.3%p 향상되었습니다.
총 불량 건수는 144건입니다.

데이터가 바뀌면? 숫자를 찾아서 수동 수정. 실수 필연적.

Quarto 인라인 코드 (안전):

이번 달 평균 수율은
96.3%이며
전월 대비
0.3%p
향상되었습니다.
총 불량 건수는
5건입니다.

데이터가 바뀌면? 자동으로 숫자도 업데이트.

Important

인라인 코드(backtick r expression backtick)는 Quarto 보고서에서 가장 강력한 기능 중 하나입니다. 숫자 한 개도 수동으로 입력하지 마세요.
