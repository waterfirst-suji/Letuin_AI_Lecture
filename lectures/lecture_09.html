4K 화면에는 픽셀이 몇 개일까?

3840 × 2160 = 8,294,400

약 830만 개의 픽셀. 각각이 빨강·초록·파랑 서브픽셀 3개로 구성. 이 2,500만 개의 불빛이 완벽하게 작동해야 한다.

불량 하나 → 불량품 하나.

이걸 시뮬레이션으로 연습할 수 있다면?

시뮬레이션이 왜 중요한가

현실의 문제

디스플레이 제조 장비 1대 = 수십억 ~ 수천억 원

실제 공정 실험은:

  • 시간이 엄청 걸린다 (수 시간 ~ 수 일)
  • 재료 비용이 막대하다 (유리 원판, 증착 재료)
  • 실패하면 복구가 어렵다

시뮬레이션의 해법

실제 공정 1회 비용:
수억 원

시뮬레이션 1회 비용:
0원 (전기세 제외)

먼저 시뮬레이션 → 검증 → 실제 적용

삼성/LG 연구소에서 매일 하는 일

디스플레이 시뮬레이션의 종류

  • 픽셀/PPI 시뮬레이터
  • 색역 시뮬레이터
  • 수율 시뮬레이터
  • 결함 시뮬레이터

목적: 해상도별 선명도 비교

PPI = √(가로픽셀² + 세로픽셀²) / 화면크기(인치)

iPhone 15 Pro:          460 PPI  → Retina 등급
Samsung Galaxy S24 Ultra: 505 PPI
일반 모니터 27":         163 PPI

인간 눈의 한계: 약 300 PPI @ 30cm 거리

목적: 색 표현 범위 비교

색역 표준 CIE 1931 커버리지 용도
sRGB 35.9% 일반 모니터
DCI-P3 53.6% 영화/전문가
Rec.2020 75.8% HDR TV

목적: 공정 변수별 수율 예측

수율(Yield) = 양품 패널 수 / 전체 생산 패널 수 × 100%

  • 삼성 OLED 패널: 95% 이상 목표
  • 초기 OLED 개발 시: 50~60%에서 시작

목적: 불량 패턴 분석

  • 픽셀 결함: 항상 켜짐(Bright Dot) / 항상 꺼짐(Dark Dot)
  • 라인 결함: 수직선/수평선 불량
  • 얼룩 결함: Mura (불균일 밝기)
  • 파티클 결함: 이물질에 의한 쇼트

library(ggplot2)

set.seed(42)
t <- seq(0, 2*pi, length.out=200)
horseshoe <- data.frame(
  x = 0.33 + 0.32*cos(t)*ifelse(sin(t)>0, 1, 0.7),
  y = 0.33 + 0.32*sin(t)
)

gamuts <- data.frame(
  Gamut = rep(c("sRGB","DCI-P3","Rec.2020"), each=3),
  x = c(0.64,0.30,0.15,  0.68,0.265,0.15,  0.708,0.170,0.131),
  y = c(0.33,0.60,0.06,  0.32,0.69,0.06,   0.292,0.797,0.046)
)

ggplot() +
  geom_polygon(data=horseshoe, aes(x=x, y=y), fill="gray20", color="gray50") +
  geom_polygon(data=gamuts[gamuts$Gamut=="Rec.2020",],
               aes(x=x, y=y), fill=NA, color="#E74C3C", linewidth=1.5) +
  geom_polygon(data=gamuts[gamuts$Gamut=="DCI-P3",],
               aes(x=x, y=y), fill=NA, color="#F39C12", linewidth=1.5) +

수율 시뮬레이션 — 공정 변수 영향

library(ggplot2)

set.seed(123)
n <- 200
sim_data <- data.frame(
  Temperature_dev = rnorm(n, 0, 1),
  Pressure_dev    = rnorm(n, 0, 1)
)
sim_data$Yield <- 97 - 1.5*abs(sim_data$Temperature_dev) -
                  1.2*abs(sim_data$Pressure_dev) + rnorm(n, 0, 0.5)
sim_data$Yield  <- pmin(pmax(sim_data$Yield, 85), 100)
sim_data$Status <- ifelse(sim_data$Yield >= 95, "양품", "불량")

ggplot(sim_data, aes(x=Temperature_dev, y=Pressure_dev, color=Yield)) +
  geom_point(alpha=0.7, size=2.5) +
  scale_color_gradientn(
    colors = c("#E74C3C","#F39C12","#27AE60"),
    values  = c(0, 0.5, 1),
    breaks  = c(87, 92, 97),
    labels  = c("87%","92%","97%")
  ) +
  theme_minimal(base_size=13) +
  labs(title="공정 변수 편차 vs 수율 시뮬레이션",
       subtitle="온도·압력 편차가 클수록 수율 하락 — 중앙(녹색)에 유지해야 95%+ 달성",
       x="온도 편차 (σ)", y="압력 편차 (σ)", color="수율")

Streamlit으로 인터랙티브 시뮬레이터 만들기

import streamlit as st
import numpy as np

st.title("🔬 OLED 수율 시뮬레이터")
st.caption("공정 변수를 조정하면 예상 수율이 바뀝니다")

col1, col2 = st.columns(2)
with col1:
    temp_tolerance = st.slider(
        "온도 허용 편차 (±°C)",
        0.5, 5.0, 2.0, 0.5
    )
    pressure_tolerance = st.slider(
        "압력 허용 편차 (±MPa)",
        0.1, 2.0, 0.8, 0.1
    )
with col2:
    particle_count = st.slider(
        "파티클 수 (ea/m²)",
        0, 100, 20, 5
    )
    align_tolerance = st.slider(
        "얼라인 허용 (μm)",
        0.1, 5.0, 1.0, 0.1
    )
base_yield = 97
yield_loss = (
    temp_tolerance     * 0.3 +
    pressure_tolerance * 0.5 +
    particle_count     * 0.02 +
    align_tolerance    * 0.4
)
predicted_yield = max(
    80,
    base_yield - yield_loss +
    np.random.normal(0, 0.3)
)

st.metric(
    "예상 수율",
    f"{predicted_yield:.1f}%",
    delta=f"{predicted_yield-95:.1f}%p vs 기준"
)

library(ggplot2)

set.seed(7)
n_sim <- 1000
mc_yield <- 96 + cumsum(rnorm(n_sim, 0, 0.1))
mc_yield  <- pmin(pmax(mc_yield, 88), 100)
monte_carlo <- data.frame(Run=1:n_sim, Yield=mc_yield)

ggplot(monte_carlo, aes(x=Yield)) +
  geom_histogram(bins=40, fill="#2E75B6", alpha=0.8, color="white") +
  geom_vline(xintercept=95,
             linetype="dashed", color="#E74C3C", linewidth=1.2) +
  geom_vline(xintercept=mean(monte_carlo$Yield),
             linetype="solid", color="#27AE60", linewidth=1.2) +
  annotate("text", x=94.3, y=80,
           label="관리\n기준", color="#E74C3C", size=3.5) +
  annotate("text", x=mean(monte_carlo$Yield)+0.6, y=80,
           label=paste0("평균\n",round(mean(monte_carlo$Yield),1),"%"),
           color="#27AE60", size=3.5) +
  theme_minimal(base_size=13) +
  labs(title=paste0("몬테카를로 수율 시뮬레이션 (n=",n_sim,"회)"),
       subtitle="무작위 시뮬레이션 1,000회 반복 — 불확실성을 통계로 처리",
       x="수율 (%)", y="빈도")

픽셀 밀도(PPI) 계산기 — Python 구현

import streamlit as st
import numpy as np
import pandas as pd
import plotly.express as px

st.title("📱 픽셀 밀도(PPI) 계산기")

col1, col2, col3 = st.columns(3)
with col1:
    width_px    = st.number_input("가로 픽셀", value=3840, step=1)
with col2:
    height_px   = st.number_input("세로 픽셀", value=2160, step=1)
with col3:
    screen_inch = st.number_input("화면 크기 (인치)", value=55.0, step=0.1)

ppi = np.sqrt(width_px**2 + height_px**2) / screen_inch

if ppi >= 400:
    grade = "Super Retina ★★★"
elif ppi >= 200:
    grade = "Retina ★★"
elif ppi >= 100:
    grade = "일반 ★"
else:
    grade = "저해상도"

st.metric("PPI", f"{ppi:.1f}", delta=grade)

# 비교 데이터
compare = pd.DataFrame({
    "디바이스": ["계산 결과","iPhone 15","Galaxy S24","4K TV 55\""],
    "PPI": [ppi, 460, 505, 80]
})
fig = px.bar(compare, x="디바이스", y="PPI", color="PPI",
             color_continuous_scale="RdYlGn")
st.plotly_chart(fig)

수율 트렌드 시뮬레이션 — 학습 곡선

library(ggplot2)

set.seed(42)
months <- 1:24
# 신규 공정 학습 곡선: 초기 저수율 → 안정화 패턴
trend  <- 75 + 20*(1 - exp(-months/8)) + rnorm(24, 0, 1.5)
trend  <- pmin(trend, 99)

yield_ts <- data.frame(
  Month = months,
  Yield = trend,
  Phase = ifelse(months <= 6, "초기 개발",
          ifelse(months <= 12, "수율 개선", "양산 안정"))
)

ggplot(yield_ts, aes(x=Month, y=Yield, color=Phase, group=1)) +
  geom_line(linewidth=1.4) +
  geom_point(size=2.5) +
  geom_hline(yintercept=95, linetype="dashed", color="#E74C3C", linewidth=1) +
  annotate("text", x=20, y=96.2, label="양산 기준 (95%)",
           color="#E74C3C", size=3.5) +
  scale_color_manual(values=c("초기 개발"="#E74C3C",
                               "수율 개선"="#F39C12",
                               "양산 안정"="#27AE60")) +
  scale_x_continuous(breaks=seq(1,24,3),
                     labels=paste0(seq(1,24,3),"개월")) +
  scale_y_continuous(labels=function(x) paste0(x,"%")) +
  theme_minimal(base_size=13) +
  labs(title="신규 OLED 공정 수율 개선 시뮬레이션 (24개월)",
       subtitle="학습 곡선(Learning Curve): 초기 75% → 24개월 후 97% 목표",
       x="경과 기간", y="수율", color="공정 단계")

시뮬레이션 vs 실제 데이터 비교

library(ggplot2)

set.seed(99)
n_compare <- 12
months_c  <- 1:n_compare

comparison <- data.frame(
  Month = rep(months_c, 2),
  Yield = c(
    90 + months_c*0.6 + rnorm(n_compare, 0, 0.3),  # 시뮬레이션
    89 + months_c*0.6 + rnorm(n_compare, 0, 1.2)    # 실제
  ),
  Type = rep(c("시뮬레이션 예측","실제 측정값"), each=n_compare)
)
comparison$Yield <- pmin(pmax(comparison$Yield, 85), 100)

ggplot(comparison, aes(x=Month, y=Yield, color=Type, linetype=Type)) +
  geom_line(linewidth=1.3) +
  geom_point(size=3) +
  scale_color_manual(values=c("시뮬레이션 예측"="#3498DB",
                               "실제 측정값"="#E67E22")) +
  scale_linetype_manual(values=c("시뮬레이션 예측"="dashed",
                                  "실제 측정값"="solid")) +
  scale_x_continuous(breaks=1:12) +
  scale_y_continuous(labels=function(x) paste0(x,"%"), limits=c(85, 100)) +
  theme_minimal(base_size=13) +
  labs(title="시뮬레이션 예측 vs 실제 수율 비교",
       subtitle="시뮬레이션 정확도가 높을수록 실제 결과와 괴리가 적다",
       x="월차", y="수율", color="", linetype="")

실습 과제 — 픽셀 밀도 시뮬레이터 구축

오늘의 미션

Streamlit + Python으로 픽셀 밀도 시뮬레이터 구축

포함할 기능:

  1. 해상도 입력 (가로 × 세로 픽셀)
  2. 화면 크기 입력 (인치)
  3. PPI 자동 계산
  4. 색상 코드 품질 등급 표시
  5. 주요 디바이스와 비교 차트

참고 공식:

PPI = √(W² + H²) / 인치

평가 기준

항목 배점
기본 PPI 계산 30점
품질 등급 판정 20점
비교 차트 추가 30점
UI/UX 완성도 20점

힌트

st.bar_chart() 또는 plotly.express.bar()를 활용하면 비교 차트를 빠르게 구현할 수 있습니다.
