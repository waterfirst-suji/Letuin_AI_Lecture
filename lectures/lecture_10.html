PART 6: 내 앱을 세상에 공개한다

만든 앱이 내 컴퓨터에만 있으면 아무 의미 없다

“이거 어떻게 보면 돼요?”

“노트북 가져올게요.” — 이 대답은 탈락이다.

URL 하나로 전 세계 어디서든 접속 가능하게 만드는 게 목표.

취업 포트폴리오도, 팀 협업 도구도, 고객 시연도 — 모두 URL 하나면 된다.

“링크 드릴게요.” 이 말이 면접관의 눈을 빛나게 한다.

배포 플랫폼 선택의 3가지 기준

1. 비용

무료인가?

  • 취업 준비생 = 무료 필수
  • 기업 프로젝트 = 유료 가능
  • 트래픽이 많아지면 → 전환

2. 기술 난이도

쉬운가?

  • HTML/CSS만 → GitHub Pages
  • Python 앱 → Streamlit Cloud
  • Full-stack → Firebase / Vercel

3. 용도

무엇을 올리나?

  • 정적 페이지 (포트폴리오)
  • 파이썬 앱 (데이터/AI)
  • AI 모델 서빙

핵심 원칙

플랫폼은 앱에 맞게 고른다. 하나가 모든 걸 해결하지 않는다.

GitHub Pages — 포트폴리오 필수

왜 GitHub Pages인가?

  • 완전 무료 (트래픽 한도 100GB/월)
  • 도메인 자동 제공: username.github.io
  • Quarto, Jekyll, Hugo 지원
  • 이력서/포트폴리오 표준 플랫폼

배포 방법

# 방법 1: /docs 폴더 사용
# Settings → Pages → Source: /docs

# 방법 2: gh-pages 브랜치
git checkout -b gh-pages
git push origin gh-pages
# Settings → Pages → Source: gh-pages branch

# 배포 주소
# https://yourusername.github.io/yourrepo

Quarto → GitHub Pages 워크플로

# 1. Quarto 렌더링
quarto render

# 2. docs 폴더에 결과물 확인
ls docs/

# 3. Git 커밋 & 푸시
git add docs/
git commit -m "Update site"
git push origin main

# 4. 자동 배포 완료!
# 약 1~2분 후 URL 활성화

취업 팁

GitHub Pages 링크를 이력서에 넣으세요. 채용담당자가 바로 확인할 수 있습니다.

Streamlit Cloud 배포 — 단계별 가이드

  • 1단계: 준비
  • 2단계: GitHub 업로드
  • 3단계: Streamlit Cloud 연결
  • 4단계: 공유
# 프로젝트 구조
my_app/
├── app.py              ← 메인 앱
├── requirements.txt    ← 의존성
└── .gitignore

# requirements.txt 예시
streamlit==1.32.0
pandas==2.0.3
plotly==5.18.0
numpy==1.24.4
# 저장소 초기화 (처음 한 번)
git init
git remote add origin https://github.com/yourname/yourrepo.git

# 파일 추가 & 커밋
git add app.py requirements.txt
git commit -m "Add Streamlit app"
git push origin main
1. share.streamlit.io 접속
2. GitHub 계정으로 로그인
3. "New app" 클릭
4. Repository 선택
5. Main file path: app.py
6. "Deploy!" 클릭

약 2~3분 후 URL 생성 완료

생성된 URL 예시:
https://yourname-yourrepo-app-xxxxx.streamlit.app

이 URL을 이력서, LinkedIn, 포트폴리오에 추가!

Firebase Studio 소개

Google의 풀스택 AI 개발 플랫폼

2024년 등장한 가장 핫한 플랫폼

Firebase Studio의 특징: - AI가 앱 아이디어를 듣고 코드 작성 - Python / JavaScript / Flutter 지원 - Firebase 인증, 데이터베이스 자동 연동 - 말하면 앱이 만들어지고 바로 배포까지

사용 시나리오:

1. "디스플레이 수율 추적 앱 만들어줘"
2. AI가 코드 생성
3. 미리보기 확인
4. Deploy 버튼 클릭
5. URL 완성

Firebase vs Streamlit 선택 기준

상황 추천
데이터 분석 앱 Streamlit
포트폴리오 사이트 GitHub Pages
풀스택 앱 Firebase
로그인 기능 필요 Firebase
AI 모델 서빙 Streamlit
React/Next.js 앱 Vercel

2024년 트렌드

Firebase Studio는 “vibe coding”의 완성형. 아이디어 → 앱 → 배포까지 수 분.

월별 배포 플랫폼 인기도 트렌드

library(ggplot2)

set.seed(7)
months_trend <- 1:12
trend_data <- data.frame(
  Month    = rep(months_trend, 3),
  Searches = c(
    80 + months_trend*0.5 + rnorm(12, 0, 2),   # Streamlit
    60 + months_trend*1.5 + rnorm(12, 0, 2),   # Firebase Studio
    90 + months_trend*0.2 + rnorm(12, 0, 2)    # GitHub Pages
  ),
  Platform = rep(c("Streamlit Cloud","Firebase Studio","GitHub Pages"),
                 each=12)
)

ggplot(trend_data, aes(x=Month, y=Searches, color=Platform, group=Platform)) +
  geom_line(linewidth=1.3) +
  geom_point(size=2.5) +
  scale_color_manual(values=c("Streamlit Cloud"="#FF4B4B",
                               "Firebase Studio"="#F9AB00",
                               "GitHub Pages"="#24292E")) +
  scale_x_continuous(breaks=1:12,
                     labels=paste0(1:12,"월")) +
  scale_y_continuous(labels=function(x) paste0(x)) +
  theme_minimal(base_size=12) +
  labs(title="배포 플랫폼 검색 트렌드 비교 (2024년 시뮬레이션)",
       subtitle="Firebase Studio 급성장 중 — 2024년 말 가장 핫한 플랫폼",
       x="", y="상대 검색량", color="플랫폼")
