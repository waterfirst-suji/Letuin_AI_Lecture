# Lecture 11-17 및 Project 1-3 정합성 메모

- 제조업 연구개발 직무를 준비하는 취업 준비생
- 반도체, 디스플레이, 배터리, 바이오 등 첨단 제조 도메인 관심자
- Gemini 유료 구독을 보유한 상태에서 한 달 동안 AI 기반 업무 방식을 학습
- 목표는 단순 AI 사용법이 아니라 데이터 분석, 시각화, 자동화, 포트폴리오 산출물 제작

## 현재 React 앱 기준 권장 흐름

현재 실제 앱은 planning 문서 일부와 주제가 다르다. 브라우저에 제공되는 React 앱 기준으로 보면 다음 흐름이 가장 자연스럽다.

| 구간 | 권장 역할 | 핵심 산출물 |
|---|---|---|
| Lecture 11 | Gemini 생태계와 API 연결 | API Key, NotebookLM, Firebase 흐름 이해 |
| Lecture 12 | 환경 변수, 보안, GitHub Actions | 안전한 API Key 관리와 CI/CD |
| Lecture 13 | 데이터 분석 자동화와 AI 인사이트 | CSV, pandas, Gemini, Streamlit 대시보드 |
| Lecture 14 | 이미지 분석 자동화 | AOI/결함 이미지 분석, Vision API 리포트 |
| Lecture 15 | 센서 데이터 예측과 알림 | 시계열 예측, 이상 감지, 알림 플로우 |
| Lecture 16 | 통합 대시보드와 포트폴리오 | 여러 분석 모듈을 취업 포트폴리오로 패키징 |
| Lecture 17 | 기술 면접과 실무 피칭 | 문제-데이터-방법-결과-임팩트 발표 구조 |
| Project 1 | 개인 포트폴리오 웹페이지 | 제조 R&D 직무 맞춤 포트폴리오 |
| Project 2 | 공정 데이터 시뮬레이터 | 수율/공정조건/결함 위험 대시보드 |
| Project 3 | 최종 통합 AI 툴 | 분석, 비전, 예측, 리포트, 알림 통합 |

## 조정한 점

- 12~17강과 Project 1~3에 강의/프로젝트 주제별 생성 이미지를 추가했다.
- 모든 신규 이미지는 각 앱의 `public` 폴더에 저장했다.
- Vite `base` 경로 아래에서도 이미지가 깨지지 않도록 `import.meta.env.BASE_URL` 기반으로 연결했다.
- 12~13강은 lecture11과 같은 이미지 카드 스타일을 추가했다.
- 14~17강은 기존 Tailwind 스타일 흐름에 맞춰 hero 직후 대표 이미지를 배치했다.
- 프로젝트 1~3은 개요 카드 다음에 프로젝트 대표 이미지를 배치했다.

## 추후 보완 권장

- planning 문서와 실제 React 앱 주제가 다른 부분은 다음 개정에서 하나로 맞추는 것이 좋다.
- 프로젝트 3은 기능을 모두 넣기보다 “제조 이상 탐지 triage assistant”처럼 명확한 제조 R&D 문제 하나로 좁히는 편이 포트폴리오 설득력이 높다.
- 각 프로젝트에는 공통적으로 `문제`, `데이터`, `방법`, `결과`, `한계`, `면접에서 설명할 포인트` 블록을 넣는 것이 좋다.
- AI 도구 자체를 강조하기보다 제조업 문제 해결의 근거와 실험적 사고를 더 앞에 두는 구성이 취업 준비생에게 더 유리하다.
