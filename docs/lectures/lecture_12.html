GitHub Actions — CI/CD 자동화

CI/CD란?

CI (Continuous Integration): 코드를 push할 때마다 자동으로 테스트

CD (Continuous Deployment): 테스트 통과 시 자동 배포

개발자 → push → GitHub →
자동 테스트 → 자동 배포 → 사용자

실제 워크플로 파일

# .github/workflows/deploy.yml
name: Deploy to Streamlit

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - run: pip install -r requirements.txt
      - run: pytest tests/

GitHub Actions의 힘

자동 테스트: - 코드 push 시 즉시 실행 - 오류 발견 시 배포 차단

자동 배포: - 테스트 통과 후 자동 배포 - 수동 배포 실수 제거

디스플레이 엔지니어링 적용

공정 데이터 분석 스크립트를 push하면 자동으로 수율 보고서가 생성되고 Streamlit 앱이 업데이트됩니다.

GitHub Secrets — 키를 안전하게 보관하는 법

Secrets 등록 방법

GitHub Repository
  → Settings
  → Secrets and variables
  → Actions
  → New repository secret

Name:  GEMINI_API_KEY
Value: AIzaSy...

workflow에서 사용

# .github/workflows/test.yml
name: Test Display App

on: [push]

jobs:
  test:
    runs-on: ubuntu-latest
    env:
      GEMINI_API_KEY: ${{ secrets.GEMINI_API_KEY }}
      ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
    steps:
      - uses: actions/checkout@v3
      - run: pip install -r requirements.txt
      - run: python -m pytest tests/ -v

왜 GitHub Secrets인가?

방법 안전성
코드에 직접 위험
.env 파일 로컬만 안전
GitHub Secrets 암호화 보관

GitHub Secrets의 특징: - AES-256 암호화 저장 - Actions 로그에서 자동 마스킹 - 저장소 접근자만 관리 가능

꿀팁

GitHub Secrets에 저장하면 Actions 로그에 *** 로 자동 마스킹됩니다. API Key가 로그에 노출될 걱정이 없습니다.

# tests/test_ppi.py
import math

def calculate_ppi(width, height, inches):
    return math.sqrt(width**2 + height**2) / inches

def test_4k_tv():
    ppi = calculate_ppi(3840, 2160, 55)
    assert 78 < ppi < 82, f"예상 PPI: ~80, 실제: {ppi:.1f}"

def test_iphone():
    ppi = calculate_ppi(2556, 1179, 6.1)
    assert 450 < ppi < 470, f"예상 PPI: ~460, 실제: {ppi:.1f}"

환경별 설정 관리 — 개발 vs 운영

여러 환경 관리하기

# 파일 구조
project/
├── .env              ← 로컬 개발용
├── .env.staging      ← 스테이징 테스트용
├── .env.example      ← 팀 공유용 (값 없음)
└── .gitignore        ← .env* 모두 제외

# .env.example (GitHub에 올리는 용도)
GEMINI_API_KEY=your_api_key_here
DATABASE_URL=your_db_url_here
DEBUG=True
# 환경별 분기 처리
import os
from dotenv import load_dotenv

# 환경 변수로 환경 구분
ENV = os.getenv("APP_ENV", "development")

if ENV == "production":
    load_dotenv(".env.production")
else:
    load_dotenv(".env")

팀 협업 시 .env 관리

팀원 A가 새로 합류했을 때:

1. git clone https://...
2. cp .env.example .env
3. .env 파일에 실제 값 입력
   (팀 채팅으로 공유)
4. python app.py

→ 코드는 같고, 키만 본인 것 사용

팀 보안 규칙

API Key는 절대 Slack/카카오톡으로 공유하지 않습니다. 암호화된 비밀 관리 도구(예: 1Password Team, Vault)를 사용하거나 직접 전달합니다.

보안 사고 대응 — Key가 유출됐다면

발견 즉시 할 일 (5분 이내)

# 1. 해당 API Key 즉시 무효화
# Google Cloud Console → API Keys → 삭제
# Anthropic → API Keys → Revoke
# GitHub → Settings → Developer settings → 삭제

# 2. 새 Key 발급
# 각 서비스에서 새 Key 생성

# 3. GitHub에서 파일 삭제 (히스토리까지)
git filter-branch --force --index-filter \
  "git rm --cached --ignore-unmatch .env" \
  --prune-empty --tag-name-filter cat -- --all

git push origin --force --all

경고

git commit에 한 번이라도 포함된 비밀은 git rm으로 삭제해도 히스토리에 남습니다. git filter-branch 또는 BFG Repo Cleaner로 히스토리까지 정리해야 합니다.
