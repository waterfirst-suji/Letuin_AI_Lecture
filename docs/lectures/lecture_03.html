자동화 시나리오 — 수동 vs AI 자동화 시간 비교

library(ggplot2)
library(tidyr)

tasks <- data.frame(
  Task       = c("커밋 메시지 작성", "코드 리뷰", "문서 업데이트", "테스트 작성", "배포 체크"),
  Manual_min = c(5, 30, 20, 45, 15),
  AI_min     = c(0.5, 5, 2, 5, 1)
)

df_long <- pivot_longer(tasks, cols = c(Manual_min, AI_min),
                        names_to = "Method", values_to = "Minutes")

ggplot(df_long, aes(x = Task, y = Minutes, fill = Method)) +
  geom_bar(stat = "identity", position = "dodge", width = 0.6) +
  scale_fill_manual(
    values = c("Manual_min" = "#C0392B", "AI_min" = "#27AE60"),
    labels = c("Manual_min" = "수동 (분)", "AI_min" = "AI 자동화 (분)")
  ) +
  theme_minimal(base_size = 12) +
  theme(
    plot.background    = element_rect(fill = "#1E2A3A", color = NA),
    panel.background   = element_rect(fill = "#1E2A3A", color = NA),
    text               = element_text(color = "white"),
    axis.text          = element_text(color = "#CBD5E0"),
    axis.text.x        = element_text(angle = 20, hjust = 1, color = "#CBD5E0"),
    panel.grid.major.y = element_line(color = "#2D3748"),
    panel.grid.minor   = element_blank(),
    panel.grid.major.x = element_blank(),
    legend.background  = element_rect(fill = "#1E2A3A", color = NA)
  ) +
  labs(
    title = "작업별 소요 시간: 수동 vs AI 자동화",
    subtitle = "하루 5개 작업 기준 — AI 도입 시 약 100분/일 절약",
    y = "소요 시간 (분)", x = "", fill = ""
  ) +
  geom_text(aes(label = paste0(Minutes, "분")),
            position = position_dodge(width = 0.6),
            vjust = -0.4, size = 3.2, color = "white", fontface = "bold")

CLAUDE.md 심화 — Hooks 설정

Hooks = 특정 이벤트 발생 시 자동 실행되는 명령어. commit 전 자동 테스트, 파일 저장 후 자동 포맷 등.

CLAUDE.md의 hooks 섹션 예시

## Hooks

### Pre-commit (커밋 전 자동 실행)
- Python 코드 품질 검사: `flake8 src/`
- 테스트 실행: `pytest tests/ -q`
- 포맷 정리: `black src/`

### Post-edit (파일 수정 후)
- 타입 체크: `mypy src/`

### Pre-push (push 전)
- 전체 테스트: `pytest tests/ -v`
- 보안 검사: `bandit -r src/`

Hooks의 실전 가치

디스플레이 수율 분석 코드에서:

  • Pre-commit: 수율 계산식 검증 (0~100% 범위 체크)
  • Post-edit: CSV 데이터 형식 검증
  • Pre-push: 전체 분석 파이프라인 동작 확인

코드 품질이 자동으로 유지된다.

취업 포트폴리오 코드에 적용하면 채용 담당자 눈에 확 띈다.

실전 스킬: /display-report 직접 만들기

전체 워크플로우:

# 1. 스킬 파일 생성
mkdir -p .claude/commands

# Claude Code에게 지시:
# ".claude/commands/display-report.md 파일을 만들어줘.
#  /display-report 명령어를 실행하면
#  data/ 폴더의 CSV를 읽고 수율 트렌드 분석 후
#  마크다운 보고서를 생성하게 해줘."

사용 예시:

# 프로젝트 폴더에서 Claude Code 실행

# 슬래시 명령어 입력

# 결과: reports/display_report_2026-04-06.md 자동 생성!
