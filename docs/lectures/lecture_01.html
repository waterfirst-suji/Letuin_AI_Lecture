바이브 코딩이란? (심화)

이건 단순히 “AI가 코드 짜주는 것”이 아니다. 패러다임의 전환이다.

1. 지시

“삼성 디스플레이 수율 분석 대시보드 만들어줘”

2. 생성

AI가 코드를 작성한다.

3. 검토 & 배포

결과물을 확인하고 수정 지시만 한다.

핵심 통찰: 코딩 능력 < 문제 정의 능력 + AI 지시 능력

디스플레이 공정 지식이 있는 당신이 AI보다 훨씬 유리한 이유가 여기 있다.

이게 가능한 이유 — AI 컨텍스트의 폭발적 성장

AI 회사들이 모델에게 내리는 비밀 지시

모든 AI 모델 뒤에는 보이지 않는 “시스템 프롬프트”가 있다. Anthropic은 이를 공개했다. 이름: Constitutional AI

원칙 1 — Harmless (해롭지 않을 것)

해를 끼칠 수 있는 행동은 거부한다. 이것이 AI가 “그건 도와드리기 어렵습니다”라고 말하는 이유.

AI 회사들이 모델에게 내리는 비밀 지시

모든 AI 모델 뒤에는 보이지 않는 “시스템 프롬프트”가 있다. Anthropic은 이를 공개했다. 이름: Constitutional AI

원칙 1 — Harmless: 해를 끼칠 수 있는 행동은 거부한다.

원칙 2 — Honest (정직할 것)

확실하지 않은 것을 확실한 것처럼 말하지 않는다. AI가 “잘 모르겠습니다”라고 말할 때, 그게 더 좋은 AI다.

원칙 3 — Helpful (도움이 될 것)

사용자의 진짜 목표를 이해하고 도움을 준다. 표면적 질문이 아닌 근본 목적을 파악한다.

이 원칙을 이해하면 AI를 더 잘 쓸 수 있다. “왜 AI가 이렇게 대답하지?”의 답이 여기 있다.

Claude Pro를 1개월 드리는 이유

이 강의 동안 Claude Pro를 마음껏 쓰세요

한 달 안에 배운 것이 평생 써먹을 도구가 됩니다.

Claude Pro의 가장 강력한 기능들:

  • Projects — 프로젝트 컨텍스트 영구 보존. 매번 설명하지 않아도 된다.
  • Artifacts — 앱/코드/문서 실시간 렌더링. 바로 결과를 본다.
  • Claude Code — 터미널에서 AI 코딩 어시스턴트 (2강에서 배움)
  • 무제한 대화 (무료 플랜은 하루 사용량 제한 있음)

현실적인 기대치 설정

AI는 도구입니다. 망치가 집을 짓지 않듯, AI가 혼자 앱을 만들지 않습니다.

당신이 방향을 잡고, AI가 실행을 도운다.

디스플레이 공정을 아는 당신의 도메인 지식 + AI의 코딩 능력 = 강력한 경쟁력

AI 도입 전후: 디스플레이 엔지니어 업무 시간 비교

library(ggplot2)
library(tidyr)

tasks <- data.frame(
  Task = factor(
    c("보고서 작성", "데이터 분석", "코드 작성", "문서 번역", "아이디어 검토"),
    levels = c("보고서 작성", "데이터 분석", "코드 작성", "문서 번역", "아이디어 검토")
  ),
  Before = c(4.0, 3.0, 5.0, 2.0, 1.5),
  After  = c(0.5, 0.5, 0.5, 0.2, 0.3)
)

df_long <- pivot_longer(tasks, cols = c(Before, After),
                        names_to = "Period", values_to = "Hours")

ggplot(df_long, aes(x = Task, y = Hours, fill = Period)) +
  geom_bar(stat = "identity", position = "dodge", width = 0.6) +
  scale_fill_manual(
    values = c("Before" = "#C0392B", "After" = "#27AE60"),
    labels = c("Before" = "AI 이전 (시간)", "After" = "AI 이후 (시간)")
  ) +
  theme_minimal(base_size = 13) +
  theme(
    plot.background    = element_rect(fill = "#1E2A3A", color = NA),
    panel.background   = element_rect(fill = "#1E2A3A", color = NA),
    text               = element_text(color = "white"),
    axis.text          = element_text(color = "#CBD5E0"),
    axis.text.x        = element_text(angle = 15, hjust = 1, color = "#CBD5E0"),
    panel.grid.major.y = element_line(color = "#2D3748"),
    panel.grid.minor   = element_blank(),
    panel.grid.major.x = element_blank(),
    legend.background  = element_rect(fill = "#1E2A3A", color = NA)
  ) +
  labs(
    title = "디스플레이 엔지니어 업무별 소요 시간 비교 (AI 전/후)",
    subtitle = "가상 시나리오 — 실제 효과는 업무 특성에 따라 다름",
    y = "소요 시간 (시간)", x = "", fill = ""
  ) +
  geom_text(aes(label = paste0(Hours, "h")),
            position = position_dodge(width = 0.6),
            vjust = -0.4, size = 3.5, color = "white", fontface = "bold")

바이브 코딩 실전 단계

Step 1: 문제를 자연어로 정의한다

“삼성 디스플레이 OLED 패널 수율 데이터를 분석하는 Python 스크립트가 필요해. CSV 파일을 읽고 이상 공정 구간을 시각화해줘.”

Step 2: AI가 코드를 작성한다

AI가 전체 코드를 작성. 에러 발생하면 “이 에러 고쳐줘”라고 말하면 된다.

바이브 코딩 실전 단계 (전체)

Step 1: 문제를 자연어로 정의 — 도메인 지식이 핵심

Step 2: AI가 코드 작성 — 에러 나면 그냥 “고쳐줘”

Step 3: 결과를 검토하고 피드백 — “수율 기준을 95%로 바꿔줘”

Step 4: 배포 — GitHub Pages, Streamlit, 사내 서버

이 4단계가 이 강의 전체 커리큘럼이다. 오늘부터 1단계를 시작한다.
