에이전트 모드: 당신이 목표 1번 말함 → AI가 필요한 모든 단계를 스스로 실행

Claude Agent가 할 수 있는 것들

에이전트 모드에서 Claude Code는 단순한 “답변 기계”가 아니다. 실제 컴퓨터 작업을 수행한다.

1. 파일 읽기/쓰기

CSV, JSON, Python, R 파일을 직접 열고 수정한다.

수율 데이터 CSV → 파싱 → 분석 결과 파일 저장

2. 코드 실행

Python, R, bash 스크립트를 직접 실행한다.

분석 코드 작성 → 바로 실행 → 결과 확인

3. 웹 검색

최신 정보를 실시간으로 검색한다.

“삼성 디스플레이 최신 채용 공고 찾아줘”

4. 도구 호출 (Tool Use)

Git, GitHub CLI, 패키지 매니저 등 외부 도구를 호출한다.

git commit, gh pr create, pip install 자동 실행

5. 멀티스텝 계획 & 실행

복잡한 목표를 단계별로 분해하고, 각 단계를 순서대로 실행한다.

“포트폴리오 웹사이트 만들고 배포해줘” → 12개 자동 실행

CLI-Anything — 모든 도구에 AI 인터페이스를

강사 직접 제작 프로젝트: “어떤 소프트웨어든 AI 에이전트 CLI로 변환”

개념: 기존 CLI 도구를 자연어로 제어

# 기존 방식: 명령어를 외워야 함
ffmpeg -i input.mp4 -vf scale=1920:1080 \
  -c:v libx264 -crf 23 output.mp4

# CLI-Anything 방식: 자연어로
cli-anything ffmpeg
> "input.mp4를 1080p로 변환해줘"
> ffmpeg 실행 중...
> 완료: output.mp4 (23.4MB)
# 디스플레이 분석 도구에 적용
cli-anything yield-tool
> "3월 2주차 수율 이상값 분석해줘"
> 데이터 로드 중...
> 이상값 3건 발견 (라인 7, 12, 19)
> 보고서 생성 완료

이 프로젝트의 의미

복잡한 CLI 도구를 배울 필요가 없다.

자연어로 말하면 AI가 올바른 명령어로 번역하고 실행한다.

디스플레이 현업에서: - 수율 분석 도구 - 공정 데이터 파이프라인 - 품질 검사 스크립트

모두 자연어로 제어 가능하게 만들 수 있다.

이것 자체가 훌륭한 포트폴리오다.

기존 사내 도구 하나를 자연어 인터페이스로 감싸는 프로젝트 → 취업 면접에서 최강 소재

에이전트 실습 시나리오 — 수율 데이터 분석 자동화

시나리오: OLED 라인 30일치 수율 데이터(CSV)가 있다. 에이전트에게 단 한 줄로 지시한다.

당신의 지시 (단 한 줄):

"yield_data.csv 분석해서 이상 구간 찾고,
 ggplot2 차트 포함한 Quarto 보고서 만들어서
 GitHub에 push해줘."

에이전트가 자동으로 수행하는 5단계:

Step 1: yield_data.csv 파일 열어서 파싱

→ 30일 × 12개 라인 = 360행 데이터 처리

Step 2: R로 통계 분석 실행

→ 평균 수율 94.2%, 이상값 3건 탐지

Step 3: ggplot2 차트 생성 (R 코드 작성 + 실행)

Step 4: Quarto 보고서 .qmd 작성 + quarto render

Step 5: git add . && git commit && git push

→ GitHub Pages 자동 업데이트

에이전트의 한계 — 알아야 잘 쓴다

에이전트가 잘 못하는 것

  • 불명확한 목표: “잘 만들어줘”는 실패한다
  • 창의적 판단: “어떤 색이 예쁜가?” → 사람이 결정
  • 도메인 상식: 수율 95%가 좋은지 나쁜지 → 당신이 판단
  • 보안 관련: 실제 내부 데이터, API 키 노출 위험
  • 긴 작업의 일관성: 100파일 이상 수정 시 맥락 손실

에이전트를 잘 쓰는 방법

  • 명확한 목표: 입력/출력/기준 명시
  • 작은 단위로 쪼개기: 한 번에 5단계 이내
  • 결과 검증: AI 출력을 맹신하지 말것
  • 도메인 지식 투입: 수율 기준, 이상값 정의는 당신이
  • 반복: 처음엔 실패해도 괜찮다. 피드백으로 개선

핵심: AI는 실행을 담당. 당신은 방향과 판단을 담당.

디스플레이 도메인 전문가인 당신이 방향을 잡으면, AI가 실행한다.

이번 주까지! 나의 디스플레이 관심 분야 소개 웹페이지

GitHub Pages에 배포하고 URL 제출

Claude Agent를 활용하세요:

"나의 디스플레이 엔지니어 포트폴리오 웹사이트를
 만들고 GitHub Pages에 배포해줘.
 내용:
 - 자기소개 (OLED/LCD 관심 분야)
 - 이 강의에서 배운 기술 스택
 - 목표: 삼성/LG 디스플레이 취업

 다크 테마, 모바일 반응형,
 GitHub Actions 자동 배포까지 설정해줘."

에이전트가 전부 알아서 한다. 당신은 내용만 확인하면 된다.

평가 기준

  • 동작 여부 (GitHub Pages URL 접속 가능): 50%
  • 내용의 진정성 (나만의 이야기): 30%
  • 디자인 완성도: 20%

제출 방법

  • 구글 폼에 GitHub Pages URL 입력
  • 마감: 5강 시작 전
  • 늦은 제출: -20점

막히면: 강의 카카오톡 채널에 에러 메시지 올리기
